# Codemap Updates — 2026-05-06

**Summary**: Updated codemaps to reflect token inactive state handling (410 Gone), millisecond timestamp precision, improved error handling, and new test coverage.

## Backend Changes

### Webhook Receiver Path (410 Gone for Inactive Tokens)
- **New repo method**: `IWebhookTokenRepository.GetByTokenIncludingInactiveAsync(Guid token)` — retrieves both active and inactive tokens for receiver path
- **Updated controller**: `WebhookController.Receive()` now:
  - Uses `GetByTokenIncludingInactiveAsync` instead of `GetByTokenAsync`
  - Wraps body read in try/catch, converts `IOException` → `BadHttpRequestException`
  - **Always** persists `WebhookRequest` (even for inactive tokens) — audit trail
  - Returns **410 Gone** for inactive tokens (signals senders to stop retrying)
  - Only notifies SSE subscribers if token is active
- **Cache**: Still uses same `"token:{guid}"` key, applies to both active and inactive

### Exception Handling
- **GlobalExceptionMiddleware** now handles:
  - `BadHttpRequestException` (400 Bad Request or 413 Payload Too Large, as per ex.StatusCode)
  - Logged with method/path context
  - Guards response write with `context.Response.HasStarted` (SSE safety)

### Database & Resilience
- **SQL Server provider**: Added `EnableRetryOnFailure(3, 2s)` in `DependencyInjection.cs`
- **ReceivedAt precision**: `HasColumnType("datetimeoffset(7)")` for millisecond precision
- **Pagination ordering**: Added `ThenByDescending(r => r.Id)` tiebreaker for deterministic results
- **Migration**: `20260506202000_PinReceivedAtPrecision` pins column type

## Frontend Changes

### Timestamp Display (Millisecond Precision)
All three surfaces now show `HH:mm:ss.SSS`:
1. **Dashboard**: `{{ token.createdAt | date:'MMM d, y, HH:mm:ss.SSS' }}`
2. **Request list**: `{{ req.receivedAt | date:'HH:mm:ss.SSS' }}` (compact, time only)
3. **Request detail**: `{{ selectedDetail()!.receivedAt | date:'MMM d, y, HH:mm:ss.SSS' }}`
