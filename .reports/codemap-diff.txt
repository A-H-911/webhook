=== Architecture Changes ===

MAJOR: Three-process architecture introduced
  - WebhookService.API — HTTP endpoints + SSE fan-out (unchanged role, slimmed DI)
  - WebhookService.StreamWorker (NEW) — drains webhook-requests Redis stream → SQL → PUBLISH sse:{tokenId}
  - WebhookService.JobsWorker (NEW) — RetentionCleanupService on 24h PeriodicTimer (single replica)

MAJOR: Redis added as infrastructure service
  - redis:7-alpine container added to docker-compose.yml
  - Stream: webhook-requests (XREADGROUP consumer group webhook-api)
  - Pub/sub: sse:{tokenId} channel (cross-process SSE fan-out)

MAJOR: DI extensions split into four focused methods
  - AddCoreInfrastructure(IConfiguration) — DB + repos + Redis multiplexer
  - AddApiInfrastructure() — publisher, token cache, SSE notifier, session store, bridge service
  - AddStreamWorkerInfrastructure() — RedisStreamConsumerService HostedService
  - AddJobsWorkerInfrastructure() — RetentionCleanupService HostedService
