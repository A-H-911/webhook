      and still persist requests for audit — only path that exercises the
      audit-persistence invariant now.

[INFRASTRUCTURE — RedisStreamConsumerService]
  + Added catch (DbUpdateException ex) when (IsForeignKeyViolation(ex))
      -> ACK + drop with logger.LogWarning
  + Added static helper: IsForeignKeyViolation(DbUpdateException)
      -> ex.InnerException is SqlException sql && sql.Number == 547
  Rationale: hard-delete + async stream pipeline can race. Without this
  handler, FK violations poison the PEL forever and block every subsequent
  webhook for the worker's process lifetime.
